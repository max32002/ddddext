DDDDEXT - 設定介面

DDDDEXT 是網頁驗證碼自動輸入機器人。

爬山想搶山屋/搶信用卡活動的登錄/掛號…都需要輸入驗證碼，透過 ddddocr 可以自動識別並輸入文字驗證碼。

搶票機制，是在比誰更快送出訂單，機器人一定比人工點擊或輸入快，一般民眾在資訊技術的落差下，是被欺凌的弱勢族群，相信分享ddddext原始碼有助於改變搶票機制，是對公眾有利益的一件事。

售票系統的「驗證碼」機制會讓一般不會寫程式的民眾更難公平地購買到預期的門票，因為機器人可以自動填入驗證碼內容。

在整個網站裡出現的「搶票」指的是「非台灣的藝文活動或車票」。

搶票程式在其他民主國家有什麼樣的標準？搶票程式只有在台灣會被處罰嗎？

台灣藝文活動的 文創法第十條 中的「 不正方式 」由於沒有明確定義，代表的是所有軟體都涉嫌違反。故在此呼籲大家，勿以身試法。

# 驗證碼網址 驗證碼圖片 selector 輸入欄位 selector 驗證碼字元數 驗證碼重設 selector
說明：
  • 「驗證碼網址」可以使用萬用字元 * 符號。
  • 「驗證碼網址」沒輸入，代表不具備功能。不知道輸入什麼，可以輸入萬用字元 * 符號，代表每一個網頁都符合。
  • 「驗證碼字元數」 欄位內容可以不輸入，不輸入代表驗證碼每次的長度是變動的。
  • 「驗證碼重設 selector」 欄位內容可以不輸入。欄位功能：當遇到OCR 失敗時, 程式要自動去按那裡, 重新產生驗證碼。有些網站是點擊驗證碼圖片重新取得，幾乎都有另外附重新取得驗證碼的按鈕。

針對不同的網頁內容，在自動填入內容時，會需要不同的設定值(selector)，selector 使用方式請參考個案教學：
https://ddddext.max-everyday.com/case/ (另開新視窗)

# Name Value Domain
如何在已經登入帳號的瀏覽器，取得 Cookies 的值？
  • 在 Windows 請按 F12 或 Ctrl + Shift + I, 在 macOS 請按 Cmd+Option+I, 即可叫出開發者工具(DevTools).
  • 在 Appliction 分頁裡的 Storage 項目中的 Cookies 裡即取得登入後 Cookie 的 Name / Value / Domain 內容值。
  • 在正常瀏覽器登入帳號後，取得 Cookie, 接著在其他視窗裡的機器人使用相同的 Cookie，這時候 2個瀏覽器(正常瀏覽器+機器人)都是使用相同的 Cookie，只要任何一個瀏覽器點了登出，這2個瀏覽器都會無法再使用該 Cookie, 因為內容已過期。
